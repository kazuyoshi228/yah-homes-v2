## 3. URL・公開ポリシー（情報衛生）

| 項目 | 仕様 |
|---|---|
| URL | `/ja/partners-jp2026/`（推測されにくいスラッグ。/partners/は使わない） |
| インデックス | `<meta name="robots" content="noindex, nofollow">` 必須 |
| サイト内リンク | 一切張らない（ヘッダー・フッター・sitemap.xml・feeds から除外） |
| 掲載しないもの | 価格・原稿料条件・戦略数値・体験団の実施状況。事実・写真・空き・条件・連絡先のみ |

- sitemap 除外: Astro の sitemap 生成対象からこのルートを除外する（filter オプション）。
- llms.txt にも載せない。
- 既存実装メモ: BaseLayout には `noindex?: boolean` prop が**実装済み**（noindex時はhreflang/x-defaultも自動抑止）。v0.1のM4は確認のみに短縮。

## 7. 空き状況（Beds24 連携）— 本仕様の核

### 7-1. セットアップ手順
1. Beds24 アカウント開設（オーナー名義。トライアル→有料は最小プラン想定・2 properties）
   - ※杉元さんから受領済みの招待コード（清川・高砂）が既存アカウントへの接続用であれば、それを優先使用（P1 MS1と共通基盤・Secret保存の承認待ち）
2. 2棟を property として登録（名称・タイムゾーン Asia/Tokyo のみ。価格は入れない）
3. Airbnb → Beds24 の iCal インポート設定
   - 運営会社に依頼: Airbnb 管理画面 → 各リスティング → 「カレンダー同期」→ iCal エクスポート URL を2棟分発行して共有（読み取り専用・権限移譲なし・5分作業）
   - Beds24 側: 各 property の iCal import に URL を登録（同期間隔は Beds24 既定・数時間毎）
4. **逆方向（Beds24→Airbnb）は設定しない**（書き込み事故防止。表示専用）

### 7-2. ウィジェット埋め込み
- Beds24 の埋め込みカレンダー（iframe）を「**空き表示のみ**」モードで2棟分設置（予約ボタン・料金表示はオフ）。
- iframe は `loading="lazy"`・高さ固定で CLS を出さない（既存 CWV 対応を壊さない）。
- 注記を併記: 「平日2泊のご提供（週末除く）」「カレンダーは数時間単位で更新されます」
- フォールバック: ウィジェットがブロックされた環境向けに「メールでお問い合わせ」リンクを直下に置く。

### 7-3. 運用ルール
- Beds24 に手動予定を入れない（**真実の源は Airbnb カレンダー**。体験団/インフルエンサーの確定日も Airbnb 側でブロック→自動反映）。
- 料金・在庫の管理には使わない（鉄則③: runtime 情報の単一ソースを守る。本件は空き表示のみ）。
