## 4.5 運営省力化機能（v0.4 追加・発注者承認済み）

### 4.5-1. 日程申請フォーム
- **位置づけ**: 既存の問い合わせフォーム（Cloud Function＋SMTP＋自動返信・稼働実績あり）の複製・項目替え。空きカレンダーを見た上で申請してもらい、日程調整の往復メールをなくす（§1 成功条件②の完成）。
- **フィールド**:

| 項目 | 必須 | 備考 |
|---|---|---|
| お名前（ニックネーム可） | ✅ | |
| メールアドレス | ✅ | 返信先・自動受付メールの宛先 |
| 媒体URL（ブログ/Instagram等） | ✅ | 発信者確認（審査を兼ねる） |
| 希望する棟 | ✅ | 清川 / 高砂 / どちらでも（選択式） |
| 第1希望・第2希望チェックイン日 | ✅ | 日付ピッカー。**月・火・水曜のみ選択可**（2泊とも平日で完結。木曜inは2泊目が金曜泊→土曜outとなるため不可） |
| 人数 | ✅ | **定員バリデーション**（清川≤7・高砂≤6をフォーム側で弾く） |
| メッセージ | 任意 | |

- **フロー**: 送信 → ①オーナーへ申請内容メール ②申請者へ自動返信「受け付けました。2〜3営業日以内にご連絡します」。オーナーの作業は可否の返信のみ。
- **メールアドレス（2026-07-21 発注者承認）**:

| 役割 | アドレス |
|---|---|
| 送信元（from） | `contact@mail.yah.homes`（既存SMTP_USER流用・SPF/DKIM実績あり） |
| オーナー通知先（to） | `kazuyoshi.yamada@bonfire.co.jp`（件名: 【パートナー申請】棟 日付〜 人数（媒体名）で問い合わせと区別） |
| 申請者への自動返信 | to=申請者入力メール・from=`contact@mail.yah.homes`・**replyTo=`kazuyoshi.yamada@bonfire.co.jp`**（返信ボタンでオーナーに直接届く） |
- ログイン不要・入力1分。フィールドはこれ以上増やさない。
- 個人情報はプライバシーポリシーの問い合わせフォーム条項でカバー（同じメール基盤）。

### 4.5-2. UTM付きリンクのコピーボタン
- **設置場所: §6「掲載時のお願い」内**（申請前のブロガーには不要な情報のため。C・2026-07-21承認）。
- 「記事にはこのリンクをお使いください」として、各棟の公式詳細ページURLに `?utm_source=partner&utm_medium=referral&utm_campaign=<媒体名など>` を付けた文字列をワンクリックコピーできるボタンを設置。
- 効果: どのブロガー記事から何人来て何件 click_airbnb したかが **GA4 に自動で分離記録**される。次回招待の優先度判断（体験団ROI）が数字で回る。
- utm_campaign は入力欄で申請者が自分の媒体名を入れてから生成（空なら partner-generic）。

### 4.5-3. 日程確定→管理会社依頼文の自動生成
- 確定日・棟を入力すると「Airbnbカレンダーのブロックをお願いします（棟・チェックイン〜チェックアウト・理由: パートナー宿泊）」の依頼メール文面を生成し、コピー or mailto で開けるミニツール。
- **設置場所はパートナーページではなく非公開**（オーナーだけが使う運用ツール。/ja/partners/ には載せない。実装は簡易ページ or ローカルスクリプトで可・実装時に相談）。

## 7. 空き状況（Beds24 API・自前描画）— 本仕様の核【v0.6でC案確定】

### 7-1. 方式
- **Beds24 API v2 から空き状況を取得し、ブランド準拠の月間カレンダーを自前描画**する。
- iCal同期・Beds24埋め込みウィジェットは**採用しない**（2026-07-21実機確認: 既存iframe booking2.php は「日付を入力して検索」する予約エンジン型で、空き日程を一覧できず用途不適合。カレンダーウィジェット案は管理会社への発行依頼が必要なため、依頼ゼロで完結するAPI案を選択）。
- 認証: 杉元さんから受領済みの招待コード（2026-07-16・清川/高砂の2本）→ refresh token に交換し **Firebase Secret Manager**（`BEDS24_REFRESH_TOKEN_KIYOKAWA` / `BEDS24_REFRESH_TOKEN_TAKASAGO`）に保存。ディスク・gitには書かない。コード失効時は杉元さんに再発行依頼（1分作業・依頼文はClaudeが用意）。

### 7-2. 実装構成
```
Beds24 API v2（在庫の真実・杉元さんアカウント）
   │ token認証・読み取りのみ
   ▼
Cloud Function bookingApi — GET /api/availability?prop={slug}
   │ ・refresh token はSecretから・クライアントに露出しない
   │ ・5分キャッシュ（表示用途に十分）
   ▼
/ja/partners/ の月間カレンダー（小さなJSアイランドがfetch→描画）
   ・空=白・塞=グレー打ち消し・モノクロ/National2のブランド準拠
   ・高さ固定でCLSゼロ（既存CWV対応を壊さない)
   ・注記併記: 「平日2泊のご提供（週末除く）」「空き状況は数分単位で更新されます」
   ・フォールバック: API不達時は「メールでお問い合わせください」を表示
```

### 7-3. 運用ルール
- **読み取り専用**（POST/書き込みAPIは呼ばない。書き込みはP1 MS3で別途設計）。
- 真実の源は Airbnb カレンダー（→Beds24に同期済み）。体験提供の確定日も Airbnb 側でブロック→自動反映。
- 料金・在庫の管理には使わない（鉄則③）。価格情報はAPIから取得しても**表示しない**。
