# yah.homes コンテンツ戦略 — 目標: オーガニック月1,000人

日付: 2026-07-14 / 親フレームワーク: magazine Blueprint v9（胴体→配管→ヘッド）
本書は yah.homes 側のコンテンツ計画。magazine を胴体、yah.homes を出力面（ヘッド⑤）として使う。

## 目標

**オーガニック検索 月1,000ユーザー**（成熟時＝公開6〜12ヶ月後）。
予約換算の期待: 月1,000流入 → 予約ページ送客10〜20% → 実予約 月2〜4組（コンテンツ起因）＋ 指名検索・AI引用の上乗せ。目標「月1組」は余裕で超える水準。

## アーキテクチャ（Blueprint準拠・優先順位のみhomes先行）

- **胴体**: magazine（CMS・品質ゲート・承認フロー・記事Firestore）
- **配管**: magazine の published & distribution=[homes] 記事 → yah.homes がビルド時に取得して /guides/ 描画（方式A/B は配管設計書で確定）
- **ヘッド**: yah.homes/guides/{lang}/{slug}（canonical は yah.homes 側＝別ブランド）
- **先行理由**: homes は具体性が高く（実在物件・福岡ローカル実測）、既にライブで計測導線（GA4・click_airbnb）が仕込み済み＝効果を測りやすい。eSIMより先に流す。

| 言語 | 成熟時流入目安 | 主戦場 |

| ja | 700〜1,200 | Google |

| zh-TW | 300〜600 | Google・Dcard/PTT |

## 記事の型（1クエリ・1躊躇・1手渡し）

- 冒頭: directAnswer（定義＋数値＋確認日＝GEO引用条件）
- 本文: この記事にしか無い実測コーパス（比較表・場合分け・距離・相場）
- 出口: 1つの躊躇を消して /booking/{kiyokawa|takasago} へ handoff
- FAQ 3問＋Article/FAQPage JSON-LD・confirmedDate
- 全意図に正直に答える（郊外を探す人にも誠実に→都心セグメントだけ手渡す）
- ローカライズ＝翻訳でなく市場語彙（zh=包棟/ko=독채）＋裏どり
