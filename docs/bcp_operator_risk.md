## 1. 前提事実（2026-07-14時点で確認済み）

- 運営委託先: **Airstar**（airstar-group.jp / airbnb-star.com）。現行2棟・管理費12%（相場15〜20%比で安い・品質に不満なし・関係良好）。
- Airstarは**自社ブランドの宿泊施設を複数運営**（AIRSTAR HOTEL SECOND・HOTEL STAR RESIDENCE Ⅰ〜Ⅲ等、福岡ほか九州・京都）。コンセプト「みんなで集まれる場所」＝**グループ滞在＝当社と同一セグメント・同一市場**。自社サイトで「公式予約ベストレート保証」を実施＝直販戦略のプレイヤーでもある。
- **Airbnbリスティングは先方名義**（ホスト: Arima Noa・スーパーホスト）。★4.77/47件（清川）・★4.67/36件（高砂）は先方アカウント上の資産。
  **Airbnbの仕様上、リスティング・レビューのアカウント間移転は不可能。** 契約書の「Airbnb資産は当社帰属」条項は、レビューについては技術的に履行できない。
- Beds24のログイン・契約も先方管理（名義は要確認）。
- 委託契約書の解約条項（予告期間・既存予約の履行義務・データ返還）: **要確認**。
- セコム等の警備駆けつけ: **未契約**。運営会社と**別軸**の物理対応オプションとして検討中（自社名義で契約すれば、運営会社に依存しない緊急対応層になる）。

## 2. 利害マップ（重要: 敵ではない。競合を兼ねるパートナー）

| 領域 | 利害 | 備考 |
|---|---|---|
| 稼働向上・広告・直販強化・単価向上 | ✅ **完全に一致** | 管理費は全チャネルの売上×%のため、当社の増収＝先方の増収。直販が削るのはOTA手数料であって先方の取り分ではない |
| 料率交渉（12→10%） | 🟡 対立するが着地可能 | 5棟化で先方の絶対額は月25万→57万超に増える |
| **同エリア・同客層の先方自社物件** | 🔴 唯一の本物の競合 | 満室時のこぼれ客・検索面の枠・グループ客の取り合い |
| 情報の非対称 | 🟡 | 先方は当社の料金・稼働・客層を全部見られる。逆は見えない |

**結論: 「いきなり切られる」確率は低い**（当社は成長中の優良クライアント・急な予約放棄は先方のスーパーホスト資格を毀損）。
現実的なリスクは急性でなく慢性の2つ:

1. **品質の漸減** — 先方ポートフォリオ拡大に伴い、清掃の良い時間枠・対応の優先度が自社物件に寄っていく
2. **学習の流出** — 当社の直販・広告の成功パターンが先方自社物件に応用される（構造上防げない。速度で勝つ領域）

## 6. 5棟化の方針と交渉議題（2026-07-14確定・同日更新）

**基本方針: 3号棟は自社運営パイロット。4・5号棟は結果を見て判断（委託/自社の両にらみ）。**
- 3号棟＝自社Airbnbアカウント・自社Beds24・構成Y（AIチャット＋本人電話）の実地試験場。レビュー資産ゼロの新棟なので失うものがない
- 判断ゲート: 運営6ヶ月後、「稼働・レビュー平均・対応事故数・本人実働時間」がAirstar棟と同等以上なら4・5号棟も自社へ。劣るなら差分を特定して委託継続
- 自社運営の粗利プレミアム＝管理費相当 約月15万/棟（年183万/棟）

| 優先 | 議題 | 位置づけ |
|---|---|---|
| **1（絶対条件）** | 委託する新棟は自社Airbnbアカウント＋共同ホスト方式 | レビュー資産の帰属。譲らない |
| 2 | Beds24の自社契約化（先方にサブアカウント付与） | データ・API主導権の恒久解。3号棟パイロットの前提でもある |
| 3 | 既存2軒の覚書（移行協力・データ提供・無断変更禁止） | |
| 4 | 料率10%（スコープ再設計: 定型連絡の自社自動化と引き換え） | **交換材料にしてよい**（名義>料率） |

**交渉の語り**: 「当社のマーケ投資は御社の増収でもある。新棟のうち1棟は自社で実験したい。4・5号棟は御社と相談したい」— 先方の取り分は2棟→将来4棟の期待が残るため関係を壊さない。

**情報衛生**: キーワード実査・広告成績・直販目標値・本書は先方に見せない。渡すのは業務分担の変更点と5棟のボリューム見込みのみ。
