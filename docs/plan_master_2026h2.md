## 2. 3本柱の関係（なぜ並行できるか）

```
B コンテンツ ──流入──▶ ②物件詳細(/properties/) ──▶ ③/book/（A柱が作る）──▶ 直販予約
C 広告      ──流入──▶ 同上（MS4で着地を自社予約に差し替え＝CVが実測可能に）
A 予約基盤  ──────── 出口を「Airbnb送客」から「自社決済」へ替える。B/Cの価値を倍化する
```

- **A柱は工学（2〜4週で完成する）。B柱は農業（月10ページ×12ヶ月）。** 混ぜずに並行。
- B/CのCTA・handoffは当面 /booking/{slug}/（現行）のまま。**MS4で全導線を一括差し替え**。

## 4. 決定済み方針（再掲・変更時は本書を更新）

- **中国語は繁体字1本**（台湾+香港先行）。hreflangは zh-TW/zh-HK/zh の3宣言で配信済み。
  簡体字は「小紅書運用開始 or GA4でzh-CN月100超」で3ページ（ホーム+物件2）のみ追加検討。
- **管理会社は維持**・5棟化契約時に10%スコープ交渉。分解外注はBATNA（見積のみ）。
- **magazineはヘッドレス胴体**: homes専売記事はmagazine表示面に出ない（feed専用）。公開は人間のみ。
- **記事の型は確定済み**（94点テンプレート）: directAnswer・確認日・目次・表・FAQ・byline（Author選択で自動）・手渡し見出し(## →)・黒背景物件カード・関連ガイド・localsクレジット。
- **デプロイ規律**: magazine=hosting+seoserverペア必須／yah.homes=記事更新後に再ビルド必要（静的）。
